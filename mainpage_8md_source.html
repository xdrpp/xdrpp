xdrpp
RFC4506 XDR compiler and message library
doc/mainpage.md
1 xdrpp
2 =====
3 
4 This package consists of an
5 [RFC4506](http://tools.ietf.org/html/rfc4506) XDR compiler, `xdrc`,
6 and an [RFC5531](https://tools.ietf.org/html/rfc5531) RPC library
7 `libxdrpp` for C++11. It takes advantage of C++ to hide most of the
8 complexity of setting up connections and cleaning up data, and
9 provides a type-safe interface that should help avoid many programming
10 errors common with other C/C++ XDR/RPC libraries. The library
11 provides both synchronous (see [srpc.h](srpc_8h.html)) and
12 asynchronous/event-driven (see [arpc.h](arpc_8h.html)) interfaces to
13 RPC. A type-safe event harness allows integration with other file
14 descriptor, timer, signal, and inter-thread callbacks (see
15 [pollset.h](pollset_8h.html)). Deterministic marshaling to
16 byte-vectors makes it easy to hash or digitally sign XDR data
17 structures (see [marshal.h](marshal_8h.html)).
18 Other features include [pretty printing](printer_8h.html), tracing
19 (set the `XDR_TRACE_CLIENT` or `XDR_TRACE_SERVER` environment variable
20 or the corresponding lower-case global `bool` values), and optional
21 integration with [autocheck](autocheck_8h.html) and
22 [cereal](cereal_8h.html) (the latter of which can, among other things,
23 translate XDR to and from JSON).

31 A high-level example gives a flavor for the library. To use RPC you
32 first define a protocol by writing a `.x` file in
33 [XDR format](http://tools.ietf.org/html/rfc4506). Then, use the
34 included XDR compiler, [`xdrc`](md_doc_xdrc_81.html), to translate
35 this file into C++ types. Finally use XDRPP's facilities to translate
36 the XDR interface into a C++ interface on both ends: The client gets
37 an object with methods corresponding to the RPC procedures, while the
38 server must implement an object with methods for each of the
39 procedures.
40 
41 Here is an example XDR source file:
42 
43 ~~~~ {.c}
44 // myprog.x
45 
46 typedef string big_string<>;
47 
48 program MyProg {
49  version MyProg1 {
50  void null(void) = 0;
51  big_string hello(int) = 1;
52  big_string goodbye(big_string) = 2;
53  } = 1;
54 } = 0x2dee1645;
55 ~~~~
56 
57 Assuming a server that has registered with rpcbind, the client can
58 invoke these RPCs as follows:
59 
60 ~~~~ {.cxx}
61 // client.cc
62 // compile as: c++ -std=c++11 `pkg-config --cflags xdrpp` -c client.cc
63 // c++ -std=c++11 -o client client.o `pkg-config --libs xdrpp`
64 
65 #include <iostream>
66 #include <xdrpp/srpc.h>
67 #include "myprog.h"
68 
69 using namespace std;
70 using namespace xdr;
71 
72 int
73 main(int argc, char **argv)
74 {
75  unique_fd fd = tcp_connect_rpc("myserver.example.com",
76  MyProg1::program, MyProg1::version);
77  srpc_client<MyProg1> c{fd.get()};
78  unique_ptr<big_string> result = c.hello(5);
79  cout << "The result of hello(5) is " << *result << endl;
80  return 0;
81 }
82 ~~~~
83 
84 A server is not much more complicated, except that it must implement
85 each of the RPC methods as methods of a C++ class. For example:
86 
87 ~~~~ {.cxx}
88 #include <xdrpp/server.h>
89 #include "xdrpp/myprog.h"
90 
91 using namespace xdr;
92 
93 class MyProg1_server {
94 public:
95  using rpc_interface_type = MyProg1;
96 
97  void null();
98 
99  unique_ptr<big_string>
100  hello(unique_ptr<int> arg);
101 
102  unique_ptr<big_string>
103  goodbye(unique_ptr<big_string> arg);
104 };
105 
106 // ... implement three methods of MyProg1 ...
107 
108 int
109 main(int argc, char **argv)
110 {
111  MyProg1_server s;
112  srpc_tcp_listener rl;
113  rl.register_service(s);
114  rl.run(); // should never return
115  return 1;
116 }
117 ~~~~
118 
119 The compiler has options (`-serverhh` and `-servercc`) to generate
120 scaffolding for `class MyProg` and its declaration.
121 
122 Note that these examples assume the server is running `rpcbind`, so
123 that the client can find the TCP port number of the RPC program on the
124 server. It is also possible to connect manually and pass file
125 descriptors for a connected socket to the constructor for
126 `srpc_client`, or to pass a `unique_fd` to an `srpc_tcp_listener` (in
127 which case the `srpc_tcp_listener` takes ownership of the file
128 descriptor).
